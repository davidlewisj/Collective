# Collective

HIPAA-compliant meeting transcription and note-taking for multi-entity healthcare organizations — Otter/Granola-class capture and notes, rebuilt around PHI-safe defaults, with no meeting bot.

**Start here:** [docs/design-spec.md](docs/design-spec.md) — the complete product design specification. The Phase 0/1 build is decomposed in [docs/engineering-backlog-phase-0-1.md](docs/engineering-backlog-phase-0-1.md).

Spec contents:

1. Product summary
2. Feature specification (capture · transcription · speaker attribution · meeting records · Claude · HIPAA · platforms/sharing · interface)
3. System architecture (components, data flow, storage)
4. HIPAA compliance matrix
5. Speaker-identification design & tradeoffs
6. Claude / MCP integration design
7. UI/UX design specification (tokens, screens, motion)
8. Open questions, risks, and build phases

This repository currently contains design documentation only — no implementation code.
